## PHASE 1 : Accueil (5 min)

### Script d'ouverture

```
"Bonjour [Prénom], merci d'avoir pris le temps.

Avant de vous montrer QALIA, j'aimerais comprendre
votre situation pour adapter la démo à VOS besoins.

Ça vous convient ?"
```

### Points à noter
- Prénom
- Organisme / structure
- Certifié Qualiopi ou pas
- Nombre de formateurs

---

## PHASE 2 : Découverte (10 min)

### Questions clés

| Question | Objectif |
|----------|----------|
| "Combien de dossiers de formation produisez-vous par mois ?" | Qualifier volume |
| "Combien de temps y consacrez-vous ?" | Quantifier douleur |
| "Quel est votre plus gros point de douleur avec Qualiopi ?" | Identifier émotion |
| "Avez-vous un audit prévu ?" | Mesurer urgence |
| "Avez-vous déjà testé ChatGPT ou autres outils IA ?" | Évaluer mindset |

### Écoute active

- Noter les MOTS EXACTS du prospect
- Ces mots = argumentaire futur
- Écouter 2x plus que parler

### Signaux forts

| Signal | Signification |
|--------|---------------|
| "40h/mois" ou plus | Douleur très haute |
| "J'ai refusé des clients" | Urgence business |
| "Audit dans X mois" | Urgence conformité |
| "Je suis seul avec ça" | Douleur émotionnelle |

---

## PHASE 3 : Démo Live (30 min)

### Principe

> **Montrer, pas expliquer. L'impact visuel > les mots.**

### Scénario type

#### Étape 1 : Contexte (2 min)
```
"Je vais créer une ingénierie complète devant vous.
On part de zéro. Vous allez voir le processus de bout en bout."
```

#### Étape 2 : Analyse des besoins (5 min)
- Montrer la maïeutique guidée
- Faire ressortir les besoins non exprimés
- Générer le Doc 02 (analyse des besoins)

#### Étape 3 : Objectifs et programme (8 min)
- Montrer la génération d'objectifs Bloom
- Montrer le plan de formation
- Souligner la cohérence automatique

#### Étape 4 : Déroulé pédagogique (10 min)
- Générer le déroulé complet
- Montrer le séquençage (15-45 min)
- Montrer l'adaptation au contexte client

#### Étape 5 : Mode Audit (5 min)
- Montrer la vérification des 32 indicateurs
- Montrer la détection codes RNCP
- Montrer le rapport de conformité

### Points d'impact à souligner

| Moment | Verbatim impact |
|--------|-----------------|
| Génération rapide | "En 1-2h au lieu de 10 jours" |
| Cohérence | "Tout se met à jour automatiquement" |
| Vérification RNCP | "Sans ça, vous l'auriez découvert en audit" |
| Personnalisation | "Chaque ingénierie est unique" |

## PHASE 4 : ROI (10 min)

### Script

```
"Maintenant, faisons le calcul pour VOUS.

Vous m'avez dit [X]h/mois à [Y]€/h.
Soit [X×Y]€/mois de temps non facturé.

Avec QALIA, vous récupérez environ 75% de ce temps.
Soit [valeur]€/mois.

QALIA coûte 297€/mois.
Votre profit net : [profit]€/mois.

L'investissement se rembourse en [payback] heures."
```

### Outils
- Utiliser le SKILL Calculateur ROI
- Montrer les chiffres en direct
- Personnaliser avec les données du prospect

---

## PHASE 5 : Questions (5 min)

### Gestion des objections

```
"Quelles questions avez-vous ?"
```

- Utiliser le SKILL Traitement Objections
- Valider, éduquer, reframer
- Ne pas argumenter

### Transition vers closing

```
"Est-ce que QALIA répond à ce que vous cherchez ?"
```

Si OUI :
```
"On peut démarrer dès maintenant. Je vous envoie le lien d'inscription."
```

Si PEUT-ÊTRE :
```
"Qu'est-ce qui vous ferait dire OUI avec certitude ?"
```

Si NON :
```
"Je comprends. Qu'est-ce qui n'a pas résonné ?"
```

## Points de Vigilance

### À FAIRE

- Adapter la démo au contexte du prospect
- Utiliser SES mots, SES douleurs
- Montrer plus qu'expliquer
- Laisser des silences (ils réfléchissent)

### À NE PAS FAIRE

- Réciter un script mécaniquement
- Noyer sous les fonctionnalités
- Parler plus que montrer
- Forcer la décision
